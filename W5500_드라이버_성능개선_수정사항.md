## 2. RX 배치 수신 구현 (핵심 성능 개선)

**목적**: 패킷 N개 수신 시 SPI 트랜잭션 횟수를 N×13회 → 5회로 줄이기

### 2-1. 기존 방식 (`w5100_rx_skb`)

패킷 1개 처리 시마다 다음 SPI 트랜잭션이 반복됨:

| 동작 | CS 횟수 |
|------|---------|
| RX_RSR read16 | 4회 |
| RX_RD read16 | 4회 |
| header readbuf (2byte) | 1회 |
| data readbuf | 1회 |
| RX_RD write16 | 1회 |
| S0_CR_RECV command | 2회~ |
| **패킷 1개 소계** | **13회~** |

→ 패킷 10개 수신 시 **130회 이상** SPI 트랜잭션 발생

### 2-2. 개선 방식 (`w5100_rx_batch`)

```
RSR 읽기(1회) → RD 읽기(1회) → 전체 버퍼 읽기(1회)
→ CPU 메모리에서 패킷 파싱 (SPI 없음)
→ RD 갱신(1회) → RECV(1회)
```

| 동작 | 기존 (N개) | 개선 후 |
|------|-----------|---------|
| RX_RSR 읽기 | N × 4회 | **1회** |
| RX_RD 읽기 | N × 4회 | **1회** |
| readbuf (헤더+데이터) | N × 2회 | **1회** |
| RX_RD 갱신 | N × 1회 | **1회** |
| RECV command | N × 2회~ | **1회~** |
| **합계** | **N × 13회~** | **~5회** |

→ 패킷 10개 기준 130회 → 5회 **(96% 감소)**

### 2-3. 구현 내용

**`w5100_priv` 구조체 변경**
```c
struct w5100_priv {
    /* 기존 필드들 */
    ...
    u8 *rx_buf;   /* RX 배치 수신용 임시 버퍼 (16KB 사전 할당) */
};
```

**`w5100_probe` 추가**
```c
priv->rx_buf = kmalloc(priv->s0_rx_buf_size, GFP_KERNEL);
if (!priv->rx_buf) {
    err = -ENOMEM;
    goto err_register;
}
```

**`w5100_remove` 추가**
```c
kfree(priv->rx_buf);
```

**`w5100_rx_batch` 신규 구현**
```c
static int w5100_rx_batch(struct net_device *ndev, int budget, bool napi)
{
    struct w5100_priv *priv = netdev_priv(ndev);
    u16 total_len, offset;
    u8 *ptr, *end;
    int rx_count = 0;

    total_len = w5100_read16(priv, W5100_S0_RX_RSR(priv));
    if (total_len == 0)
        return 0;

    if (total_len > priv->s0_rx_buf_size)
        total_len = priv->s0_rx_buf_size;

    offset = w5100_read16(priv, W5100_S0_RX_RD(priv));

    if (w5100_readbuf(priv, offset, priv->rx_buf, total_len))
        return 0;

    ptr = priv->rx_buf;
    end = priv->rx_buf + total_len;

    while (ptr + 2 <= end && rx_count < budget) {
        u16 pkt_total = get_unaligned_be16(ptr);
        u16 rx_len;
        struct sk_buff *skb;

        if (pkt_total < 2) {
            /* 진짜 corruption: 전체 버리기 */
            netdev_err(ndev, "rx: corrupted packet header: pkt_total=%u\n", pkt_total);
            ptr = end;
            break;
        }
        if (ptr + pkt_total > end)
            break;  /* 배치 끝에 걸친 패킷 — 다음 인터럽트에서 처리 */

        rx_len = pkt_total - 2;
        skb = netdev_alloc_skb_ip_align(ndev, rx_len);
        if (unlikely(!skb)) {
            ndev->stats.rx_dropped++;
            ptr += pkt_total;
            continue;
        }

        skb_put(skb, rx_len);
        memcpy(skb->data, ptr + 2, rx_len);
        skb->protocol = eth_type_trans(skb, ndev);
        ndev->stats.rx_packets++;
        ndev->stats.rx_bytes += rx_len;

        if (napi)
            netif_receive_skb(skb);
        else
            netif_rx(skb);

        ptr += pkt_total;
        rx_count++;
    }

    w5100_write16(priv, W5100_S0_RX_RD(priv), offset + (u16)(ptr - priv->rx_buf));
    w5100_command(priv, S0_CR_RECV);

    return rx_count;
}
```

**호출부 교체**
```c
/* workqueue 경로 */
static void w5100_rx_work(struct work_struct *work)
{
    struct w5100_priv *priv = container_of(work, struct w5100_priv, rx_work);
    w5100_rx_batch(priv->ndev, INT_MAX, false);
    w5100_enable_intr(priv);
}

/* NAPI 경로 */
static int w5100_napi_poll(struct napi_struct *napi, int budget)
{
    struct w5100_priv *priv = container_of(napi, struct w5100_priv, napi);
    int rx_count = w5100_rx_batch(priv->ndev, budget, true);

    if (rx_count < budget) {
        napi_complete_done(napi, rx_count);
        w5100_enable_intr(priv);
    }
    return rx_count;
}
```

### 2-4. Partial Packet 처리 (핵심 엣지케이스)

`RX_RSR`은 바이트 단위로 업데이트되므로 패킷 수신 중간에 읽히면 버퍼에 패킷 일부만 있을 수 있음.

| 조건 | 처리 | 이유 |
|------|------|------|
| `pkt_total < 2` | `ptr=end`, break | 진짜 corruption |
| `ptr + pkt_total > end` | `break`만 (RX_RD 유지) | Partial packet, 다음 인터럽트에서 재처리 |

> **주의**: `ptr + pkt_total > end` 시 `ptr=end`로 설정하면 RX_RD가 잘못 이동해 다음 읽기에서 패킷 중간부분을 헤더로 읽는 연쇄 오류 발생 (경험한 버그)

## 5. 미구현 (추가 개선 가능 항목)

### 5-1. `w5100-spi.c` SPI CS 토글 개선

현재 레지스터 읽기/쓰기 시 `spi_write_then_read`를 두 번 호출하여 CS를 2회 토글.
`spi_sync_transfer`로 하나의 트랜잭션으로 합치면 CS 토글 횟수 50% 감소.

### 5-2. `w5100_command` SEND 폴링 제거

TX 명령 후 완료를 폴링하는 구조를 제거하여 TX 레이턴시 개선.

## 6. 참고사항

- W5500의 최대 SPI 클럭: 20MHz (DTS 설정값)
- 이론 최대 처리량: 20Mbps = 2.5 MB/s
- 현재 달성: ~1.2 MB/s = **이론값 대비 약 48%**
- 나머지 손실은 SPI 프레임 오버헤드, CS 토글, 드라이버 처리 시간
